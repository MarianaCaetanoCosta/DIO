Quiz - Entendendo Métodos Java

Tendo o método a seguir, qual não é uma sobrecarga válida?

public int avaliar()

public int avaliar (double i)


Sobre métodos, marque o que for VERDADE:

Todo método deve ter um nome


Sobre métodos, marque o que for INCORRETO:

Os parênteses são opcionais na definição de métodos


Um método deve ser público, sem retorno, se chamar xpto, tem um int como parâmetro  e ter corpo vazio. 
Baseado nessa especificação, qual é a definição correta?

public void xpto (int i) {}


Em relação a assinatura de um método, marque o CORRETO:

É constituida do nome e lista de parâmetros do método.


Avalie o método a seguir e marque a alternativa CORRETA:

public void verifica(int i) {
   if (i > 1) {
     return “Maior”;
   } else {
     return “Menor”;   
   }
}

Ocorre um erro de compilação, pois o método "void" não tem retorno.


A partir do método a seguir, marque a alternativa CORRETA:

public int verifica(int i) {...}


"public int verifica (int i) { return 0;}" é uma implementação valida


Tendo o método a seguir, marque o que for INCORRETO:

public double calcularTotalNota() {…}

Ele pode retornar o valor do tipo texto


Sobre métodos, marque o que for INCORRETO:

Métodos só podem ser utilizados a partir de classes, de forma estática.


Avalie o método a seguir e marque a alternativa INCORRETA:

public String verifica(int idade) 
   if (idade > 18) {
     return “Maior de idade”;
   } else {
     return “Menor de idade”;   
   }


O corpo do metodo está definido corretamente
